## はじめに

毎朝、株の注文前に「これ、今日買っていいやつ？」と迷ってませんか？

PER見て、配当利回り調べて、チャート確認して……気づいたら30分経ってて、結局「なんかいい感じだから買う」という感情買いに落ち着く。

自分もそうだった。

だから作った。

**「今日買うか、待つか、見送るか」を自動で答えてくれるシステム**を。

---

## このシステムでできること

- 監視銘柄を毎朝自動で取得・採点（yfinance使用）
- VIXと日経平均から市場の「地合い」を自動判定
- 100点満点のスコアで銘柄を格付け
- 「今日の水準で1株」「前日比-5%で買い」「今は見送り」など、**具体的なアクションを出力**
- 結果をGoogle SheetsとHTMLダッシュボードに自動反映
- GitHub Actionsで**平日08:00 JSTに全自動実行**

**使う技術:** Python / yfinance / pandas / Google Sheets API / GitHub Actions

## なぜ作ったか（背景）

自分の投資スタイルはシンプル。

```
SBI証券のS株（ミニ株）で1株ずつ、成行注文で淡々と積み立てる。
基本的に売らない。配当を積み上げて幸福度を上げる。
```

この「淡々と」がむずかしい。

- 急落したとき：「やばい、売るべき？」
- 急騰したとき：「乗り遅れた、今から買う？」
- 何も動いてないとき：「今日は何か買うべき？」

毎日こういう判断をしていると疲れる。感情が入る。ルールを守れなくなる。

だから**ルールをコードに落とし込んで、判断を自動化**した。

## スコアリングの設計（100点満点）

ここが一番こだわった部分。**「感情ではなくロジックで銘柄を評価する」**ための採点基準。

### 配当の質（35点）——最重視

長期ガチホの幸福度は配当に直結するので、**配点を一番大きくしている**。

| 指標 | 配点 | 基準 |
|------|------|------|
| 連続増配年数 | 12点 | 0年=0点, 5年=6点, 10年以上=12点 |
| 過去10年の減配回数 | 10点 | 0回=10点, 1回=7点, 2回=4点, 3回以上=0点 |
| 配当性向 | 8点 | 30-50%=8点（理想）, 50-60%=5点, 60-70%=2点 |
| 配当利回り | 5点 | 2-3%=2点, 3-4%=4点, 4%以上=5点 |

**配当性向の考え方:**
- **30-50%：余力たっぷり。増配できる体力がある**
- 70%超：利益のほとんどを吐き出している。危険ゾーン

### 業績の成長力（30点）

配当は業績が伸びないと増配できない。**ガチホの含み益にも直結する**。

| 指標 | 配点 | 基準 |
|------|------|------|
| 売上高成長率（3年CAGR） | 10点 | 0%以下=0点, 5%未満=4点, 10%未満=7点, 10%超=10点 |
| 営業利益成長率（3年CAGR） | 10点 | 同上 |
| ROE | 5点 | 5%未満=0点, 8%未満=2点, 12%未満=4点, 12%超=5点 |
| 営業利益率 | 5点 | 5%未満=0点, 10%未満=2点, 15%未満=4点, 15%超=5点 |

### 財務健全性（20点）

**ガチホで安心して眠れるかどうか**。

| 指標 | 配点 | 基準 |
|------|------|------|
| 自己資本比率 | 8点 | 20-40%=2点, 40-60%=5点, 60%超=8点 |
| D/Eレシオ | 6点 | 0.5未満=6点, 0.5-1.0=3点, 1.0超=0点 |
| FCF（直近3年連続プラス） | 6点 | 3年=6点, 2年=3点, 1年以下=0点 |

### 割安度（15点）

同じ良い銘柄なら安く買いたい。

| 指標 | 配点 | 基準 |
|------|------|------|
| PER | 8点 | 業種平均比で採点 |
| PBR | 4点 | 1.0未満=4点, 1.0-2.0=2点, 2.0超=0点 |
| ミックス係数（PER×PBR） | 3点 | グレアム基準。22.5以下が割安 |

## コードのキモ：地合い判定

単純な銘柄スコアだけじゃなく、**「今の市場全体の状況」を加味する**のがこのシステムの特徴。

```python
def get_market_context() -> MarketContext:
    vix_history = yf.Ticker("^VIX").history(period="1mo")["Close"].dropna()
    nikkei_history = yf.Ticker("^N225").history(period="3mo")["Close"].dropna()

    vix = float(vix_history.iloc[-1]) if not vix_history.empty else None
    nikkei_5d = ((float(nikkei_history.iloc[-1]) / float(nikkei_history.iloc[-6])) - 1) * 100

    if vix >= 30 or nikkei_5d <= -8:
        return MarketContext(regime="恐怖", buy_dip_bonus=8,
                            note="恐怖局面。高品質株は押し目を買う日")
    if vix >= 25 or nikkei_5d <= -5:
        return MarketContext(regime="弱気", buy_dip_bonus=5,
                            note="弱気相場。前日比-3%~-5%の押し目を狙う")
    # ... 中立・楽観も同様
```

**恐怖局面（VIX≧30）のとき、高品質な銘柄には+8点のボーナスが入る。**
バフェットの「皆が恐怖のとき貪欲に」をコードで実装したイメージ。

## 出力されるアクション

採点結果をもとに、以下の**具体的なアクションが自動で決まる**：

| アクション | 意味 |
|-----------|------|
| 今日の水準で1株 | 条件全揃い。今すぐ1株買う |
| 前日比-3%で買い | 恐怖/弱気相場で押し目を前倒し待機 |
| 前日比-5%で買い | もう一段の押し目を待ってから買う |
| まあ1株エントリー可 | 強い押し目ではないが少額なら許容 |
| 押し目を待つ | 品質は高いが、まだ下げが足りない |
| 今は買わない（高値圏） | 高値追い禁止 |
| 条件待ち | 監視継続 |
| 見送り | 今日は触らない |

## 使用技術・環境

```
Python 3.10+
yfinance 0.2.x          ← 日本株のOHLCVとファンダメンタルズを無料取得
pandas 2.2.x            ← データ処理
google-api-python-client ← Google Sheetsに自動書き込み
google-auth-oauthlib    ← OAuth認証
GitHub Actions          ← 平日08:00 JSTに全自動実行
```

**yfinanceは無料・APIキー不要**で日本株（東証）のデータが取れるのが最高。
配当履歴、財務諸表（損益計算書・貸借対照表・CF計算書）まで一括取得できる。

## 実装で詰まったポイント

正直なところも書いておく。

### 1. Google Sheets認証がつらかった

- OAuthクライアントJSONの取得
- ローカルでの初回認証フロー
- トークンの保存場所とGit管理からの除外

など、**最初はつまづきポイントが多かった**。

### 2. yfinanceの仕様変更

- ドキュメントと実際の挙動が微妙に違ったり、エラーが返ってくるケースもある
- **失敗時のリトライや、「この銘柄は諦めてスキップ」の設計が必要**

### 3. スコアリングの重み付け

- 最初は配当利回りに寄せすぎて、「高配当だけど業績ボロボロ」な銘柄が上位に来てしまった
- 何度かテストしながら、**「配当の質＞成長＞財務＞割安」のバランスに落ち着いた**
